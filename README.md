# Auto Sort — автоматическое перераспределение остатков WB

Бот читает задания из Google-таблицы и автоматически создаёт в Wildberries заявки на перемещение товаров между складами. Отслеживает доставку и помечает выполненные задачи.

## Что делает бот

- **Читает таблицу** «Задания» раз в 5 минут + специально в 00:00 / 09:00 / 18:00 МСК (когда открываются квоты складов)
- **Находит строки** со статусом «Создан»
- **Отправляет заявку в WB** на перераспределение остатков
- **Пишет в таблицу**: статус «В очереди бота», дата выполнения, комментарий при ошибке
- **Проверяет доставку** каждое утро в 07:00 МСК — когда товар реально приехал на склад, ставит «Выполнен ботом»

## Что ожидать по срокам

| Этап | Время |
|---|---|
| От добавления строки до отправки заявки в WB | до 5 минут |
| Обработка заявки Wildberries и физическое перемещение | 1–3 дня |
| Автоматическая отметка «Выполнен ботом» после приезда товара | на следующее утро (07:00 МСК) |

## Работа менеджера

### Таблица «Задания»

Заполняй колонки A–G и H (статус = «Создан»). Дату выполнения, требует-реакции и комментарий **бот пишет сам** — менеджеру их трогать не надо.

**Важно:**
- Добавлять новые строки **только в конец** таблицы
- Менять любые поля в уже существующих строках (например, статус)
- **Не удалять** строки — это ломает отслеживание. Если задача больше не нужна — поменяй статус на «Отменен по дедлайну»
- **Не вставлять** строки в середину — по той же причине
- **Не сортировать** таблицу

### Админ-панель

Доступна по адресу сервера (указан в `.env` → `DOMAIN`). Пароль — `ADMIN_SECRET_KEY` из `.env`.

Три раздела:
- **Дашборд** — статистика (неделя / месяц / всё время) + состояние бота (🟢 Работает / ⏸ Остановлен / 🔴 Ошибка) + индикатор сессии WB. Тут же кнопка «▶ Запустить / ⏸ Остановить» и «Последняя обработка: ...»
- **Задания** — таблица с фильтром: можно отобрать только строки со статусом N или только «Требует внимания»
- **Настройки** — форма обновления куки WB, изменение интервала опроса (1-60 мин), кнопка «Проверить сессию», ручной запуск обработки

Если на дашборде видишь **жёлтый баннер** `⚠️ N строк требует внимания` — нажми на него, перейдёшь на отфильтрованный список. В колонке «Коментарий» будет объяснение что пошло не так.

В таблице «По менеджерам» для каждого менеджера видны все 7 метрик: заданий, штук, % успеха, в работе, отменено, топ складов отгрузки/прихода.

### Что ещё можно сделать из админки

- **«⏸ Остановить / ▶ Запустить»** — пауза автоматических задач (например, перед отпуском или при техработах). Ручной запуск продолжает работать.
- **«✕» рядом с красным сообщением об ошибке** — очистить информацию о последней ошибке (после того как разобрался)
- **Смена интервала опроса таблицы** (Настройки → «Интервал опроса таблицы», 1-60 мин) — применяется сразу без перезапуска
- **«Сбросить все сессии»** (Настройки → «Безопасность») — вылогинить всех кто сейчас залогинен. Нужно при утечке пароля или если сотрудник ушёл.

### Обновление куки WB

Индикатор сессии на дашборде бывает трёх цветов:
- 🟢 **Активна** — всё работает
- 🟡 **Неизвестно** — нажми «Проверить сессию» в настройках
- 🔴 **Истекла** — нужно обновить куки

**Как обновить** — смотри видео-инструкцию в проекте (в папке Instructions).
В админ панель **обязательно** нужно вводить куки именно такого типа, как на видео. Cookie Header **обязательно** должен быть указан WBTokenV3.

## Что делать если что-то пошло не так

**Бот не обрабатывает задания:**
1. Зайди в админку → проверь индикатор сессии
2. Если 🔴 — обнови куки (см. видео)
3. Если 🟢 и всё равно не обрабатывает — напиши разработчику

**Задания висят в «В очереди бота» больше 3 дней:**
- Нормально до 72 часов
- Если дольше — проверь в Wildberries через личный кабинет состояние перемещения
- Возможно WB не смог выполнить (нет товара на складе-источнике, квоты были исчерпаны)

**Весь список ошибок с объяснениями** бот пишет в колонку «Коментарий».

## Что внутри

Для тех, кто будет поддерживать код, короткий обзор:

- **Python 3.12 + FastAPI** — админ-панель
- **PostgreSQL** — хранит зашифрованные куки WB, записи об отправленных заявках (`task_deliveries`), состояние бота (`app_state`)
- **APScheduler** — 5 cron/interval-задач: опрос таблицы, окна квот, проверка куки, проверка доставок, dead letter scan
- **Docker Compose** со всеми сервисами: app, db, nginx (TLS через Let's Encrypt), certbot (auto-renewal), backup (pg_dump раз в сутки)
- **Тесты**: 81 unit-тест (parser, client, auth, lk_stocks, retry, run_lock, task_processor helpers)
- **Безопасность**: CSRF, rate limiting, session revocation, circuit breaker, secure cookies при HTTPS
